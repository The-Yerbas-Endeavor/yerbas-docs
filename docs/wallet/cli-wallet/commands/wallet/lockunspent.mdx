# lockunspent

_lockunspent unlock ([{"txid":"txid","vout":n},...])_

Updates list of temporarily unspendable outputs.
Temporarily lock (unlock=false) or unlock (unlock=true) specified transaction outputs.
If no transaction outputs are specified when unlocking then all current locked transaction outputs are unlocked.
A locked transaction output will not be chosen by automatic coin selection, when spending yerbass.
Locks are stored in memory only. Nodes start with zero locked outputs, and the locked output list
is always cleared (by virtue of process exit) when a node stops or fails.
Also see the listunspent call

```bash
# List the unspent transactions
yerbas-cli listunspent

# Lock an unspent transaction
yerbas-cli lockunspent false "[{\"txid\":\"a08e6907dbbd3d809776dbfc5d82e371b764ed838b5655e72f463568df1aadf0\",\"vout\":1}]"

# List the locked transactions
yerbas-cli listlockunspent

# Unlock the transaction again
yerbas-cli lockunspent true "[{\"txid\":\"a08e6907dbbd3d809776dbfc5d82e371b764ed838b5655e72f463568df1aadf0\",\"vout\":1}]"
```

## Arguments

| Argument       | Type      | Description                                                 |
| -------------- | --------- | ----------------------------------------------------------- |
| unlock         | (boolean) | The number of transactions to return                        |
| "transactions" | (object)  | A json array of objects. Each object the txid (string) vout |

```bash
Arguments:
1. unlock            (boolean, required) Whether to unlock (true) or lock (false) the specified transactions
2. "transactions"  (string, optional) A json array of objects. Each object the txid (string) vout (numeric)
     [           (json array of json objects)
       {
         "txid":"id",    (string) The transaction id
         "vout": n         (numeric) The output number
       }
       ,...
     ]
```
